# PPT 演讲者备注生成 Skill

## 第一步：收集必要信息

在生成备注之前，先读取用户提供的幻灯片内容（.pptx 文件优先，其次是粘贴文字），然后主动询问以下信息：

**必须主动询问的（用户未说明时不要假设）：**
- **汇报场合 / 听众对象**：面向领导、专家评审、客户、内部团队、学生/公众，还是竞赛答辩？
- **总汇报时长**：大约多少分钟？（影响每页备注密度）

**有说明就用，没说明走默认：**
- 每页预计时长：以总时长平均分配，或按内容比重灵活调整
- 语言：中文

**询问方式**：简洁地一次性问清楚，不要逐项询问，不要等用户回答才读取内容。可以先确认已读取内容，再一次问两个问题。

---

## 第二步：核心生成规则

### 2.1 内容忠实性（最高优先级）

所有事实性内容必须来自幻灯片原文，包括：
- 数字、比例、金额、时间、年份
- 人名、机构名、项目名、产品名、系统名
- 专业术语、政策名称、技术名称
- 结论、成果、问题、目标、任务、路径等核心表述

**可以做的**：基于原文进行逻辑解释、语言衔接、表达转换和适度概括。  
**不可以做的**：凭经验补充原文没有的数据、案例或结论。

### 2.2 根据听众调整表达

| 听众类型 | 表达重点 |
|----------|----------|
| 领导 | 简洁有重点，突出结论、价值、下一步 |
| 专家评审 | 严谨逻辑完整，突出依据、方法、创新 |
| 客户 | 易懂有代入感，突出问题、方案、效果 |
| 内部团队 | 务实直接，突出进展、问题、协同 |
| 学生或公众 | 通俗生动，减少术语堆砌 |
| 竞赛或答辩 | 紧凑有冲击力，突出亮点和差异化 |

### 2.3 尊重页面结构

- 多个并列板块、流程步骤、层级结构 → 按原有结构逐一展开，不合并、不遗漏
- 内容简单的页面 → 简短概括，不强行加戏

---

## 第三步：语言风格要求

### 口语化但不随意
备注是演讲者直接照着讲的口头稿，适合正式场合朗读，避免论文腔、八股腔。

### 禁用的表达（严格避免）
```
这一页……  /  本页……  /  页面中……  /  页面文字中提到……
PPT 展示了……  /  幻灯片上可以看到……
左侧案例是……右侧案例是……（机械版面说明）
```

### 推荐的表达方式
```
接下来重点汇报……
这里我想强调的是……
我们可以从两个场景来看……
第一个场景是……第二个场景是……
从这个结果可以看到……
这也说明后续优化需要重点关注……
```

允许根据图示方位进行口头引导（"左边这条线代表……"），但要像现场讲解，不是描述版式。

### 禁用空泛套话
除非原文明确出现，否则不要使用：
> 意义重大、成效显著、全面赋能、深度融合、夯实基础、提质增效、打造标杆、构建新格局

### 主语选择
根据内容类型自动判断，不固定"本项目"或"我们"：

| 内容类型 | 推荐主语 |
|----------|----------|
| 项目汇报 | 本项目、我们 |
| 产品介绍 | 这款产品、该方案 |
| 教学课件 | 这个概念、这一部分 |
| 年终总结 | 这一阶段、团队、我们 |
| 学术报告 | 本研究、本文、我们 |
| 方案宣讲 | 这套方案、整体思路、我们 |

## 第四步：长度与节奏控制

### 按页面类型控制备注长度

| 页面类型 | 备注写法 |
|----------|----------|
| 封面页 | 简短开场，说明汇报主题、背景或表达感谢 |
| 目录页 / 结构页 | 简要说明汇报框架 |
| 章节分隔页 | 一两句过渡，不展开正文 |
| 背景页 / 问题页 | 适当展开背景、问题来源和讲述必要性 |
| 目标页 / 思路页 | 讲清楚目标指向、总体逻辑和展开路径 |
| 核心内容页 | 重点展开，解释关键概念、方法、机制 |
| 流程图 / 架构图页 | 按图示逻辑讲解，突出主线和关键节点，不逐字念图 |
| 数据图表页 | 先说图表看什么，再说趋势或结论，不逐项报数 |
| 表格页 | 概括分类逻辑、重点差异和总体结论，不逐行朗读 |
| 案例页 | 按背景、做法、结果或价值展开，避免流水账 |
| 成果页 / 总结页 | 提炼重点，强调价值、变化或结论 |
| 结束页 | 简短收束，回扣主题，自然表达感谢 |

### 按总时长控制备注密度

| 总时长 | 密度要求 |
|--------|----------|
| 5 分钟以内 | 高度精炼，只讲主线和关键结论 |
| 5～10 分钟 | 适度展开重点页，过渡页从简 |
| 10～20 分钟 | 核心页充分展开，注意前后逻辑衔接 |
| 20 分钟以上 | 可加入更多解释和过渡，但不得脱离原文 |

如果用户提供了每页预计时长，优先服从每页时长安排。

## 第五步：输出方式

根据使用环境不同，输出方式分三种情况：

### 情况 A：Claude for PowerPoint 插件

用户在 PowerPoint 中直接调用 Claude，Claude 可以访问当前打开的文件。此时：
- **无需文件上传和下载**，直接将备注写入当前演示文稿对应幻灯片的备注区
- 写完后告知用户已完成，以及写入了哪几页

### 情况 B：上传 .pptx 文件（Claude Code / agent 环境）

用户将文件上传到对话中。此时：
- 配合 pptx skill（`/mnt/skills/public/pptx/SKILL.md`）将备注逐页写入 notes 区域
- 写入完成后告知用户，说明文件保存位置或提供下载方式（根据当前运行环境决定，不统一预设）

### 情况 C：仅粘贴文字内容

用户没有上传文件，只粘贴了幻灯片文字。此时在对话中逐页输出，格式如下：

```
【第 N 页】页面标题（如有）

备注正文……

---
```

用户可自行复制粘贴到 PPT 备注区，或在后续上传文件时写入。

---

## 注意事项

- 不要主动评价 PPT 设计质量或提出修改建议，除非用户明确要求
- 不要在备注中解释"这是演讲者备注"之类的元说明
- 如果某页内容过于简略无法判断类型，简短处理即可，不要强行脑补
