## 一、任务识别与路由

收到任务后，**先按下表路由，再读取 `references/modes.md` 中的对应模式执行**：

| 模式    | 名称    | 一句话说明                        | 触发关键词                     |
| ----- | ----- | ---------------------------- | ------------------------- |
| **A** | 代写生成  | 从零或素材出发，通过"生成→评审→修改"迭代闭环生成文章 | "代写""帮我写""起草"             |
| **B** | 润色改写  | 保留内容逻辑，注入风格，消除 AI 腔          | "润色""改写""让它更像我的风格"        |
| **C** | 风格迁移  | 将文章从文体 A 转换为文体 B             | "迁移""把这篇变成XX风格"           |
| **D** | 建库/更新 | 从新样本提炼风格指纹，写入档案              | "分析风格""加入/更新风格库"          |
| **E** | 反馈学习  | 对比 AI 稿与用户定稿，提取差异，更新规则       | "对比学习""我改了""学习这个版本""更新规则" |

文体/模式不明确时，用一句话确认：*"请确认：①文体类型（学术论文/公众号/课题方案/工作方案），②任务模式（代写/润色/迁移）。"* 确认后直接执行，不再追问。

## 二、双智能体机制（模式 A 代写专用）

**长文代写（≥2000字）默认启用双智能体流程。** 短文（<2000字）仍用单智能体 + 标点自检。用户可通过"跳过评审"关键词手动关闭。

**核心理念：** 将内容生成与内容评审交给两个独立的子智能体，形成对抗性迭代闭环。

| 智能体 | 职责 | 接收内容 |
|---|---|---|
| **生成智能体** | 按作者风格规则撰写正文 | style-core + 文体档案 + **声音参照段落（必须）** + **改写对照（必须）** + 写作任务 + 规划 |
| **评审智能体** | 审查内容质量 + 声音偏差 | 评审标尺（含V0维度）+ **声音参照段落（必须）** + 草稿全文 |

**关键原则：**

- 主控在调度前必须完成声音地基加载（第2.5步）：从样本库抽取 2-3 个原文段落 + 从改写对照库选 8-10 对，直接嵌入两个智能体的 prompt
- 评审智能体不加载风格规则文件，接收的声音参照是**原文段落**，用于 V0 声音偏差对比，不是规则检查
- 评审智能体冷启动，不共享生成智能体上下文，天然具备外部视角
- 最多迭代 2 轮，超限交由用户决策

## 三、通用规则（所有模式适用）

### ⛔ 输出前必须执行的风格合规扫描

**每次生成或润色完成后，在输出给用户之前，必须执行完整的风格合规扫描（S1-S5），不得跳过。**

详细扫描清单定义在 `references/modes.md` 模式 A 第6步中，覆盖五个维度：

- **S1 标点扫描**：冒号、破折号、分号、括号逐个判断必要性
- **S2 禁忌句式扫描**："不是…而是…"、万能修饰词、编号过渡词
- **S3 指示代词扫描**："这/这一/这个/它"频次控制
- **S4 内部指令泄漏扫描**：规则内容不得出现在正文中
- **S5 子节同构扫描**：连续同级子节不得结构完全相同

核心判断标准不变：**这里用逗号或句号能否同样把意思说清楚？能说清楚就不换特殊标点。**

**扫描必须产出结构化报告**（各维度的发现数和修改数），报告为空则对前三段抽检确认。据报告就地修改全文，改完才能输出。**这是输出的最后一道门。**

> ⚠️ 若文体为**公众号**，完成风格合规扫描后，还需执行 `references/style-wechat.md` 第八节"输出前自检（公众号专用）"中的 L4 活人感自检和 HKR 内容质量自检，三项全部通过再输出。

---

### 输出后反馈入口

**完成模式 A/B/C 的正文输出后，在末尾另起一行，追加以下提示：**

> *如需将您对本文的修改纳入风格学习，请发送修改后版本并附言"对比学习"。*

此提示是反馈学习闭环的入口，不得省略。若用户已明确说"不需要学习"，可跳过。

## 四、资源读取规则

### 4.1 三层读取顺序（每次必须遵循）

**所有写作任务的读取顺序：core → 文体档案 → 样本（按需）**

| 步骤   | 读取文件                          | 读取范围                       |
| ---- | ----------------------------- | -------------------------- |
| ① 必读 | `references/style-core.md`    | 完整阅读"思维倾向参考"、"语感指纹"和"稳定禁忌" |
| ② 必读 | 对应文体的 `references/style-*.md` | 完整阅读                       |
| ③ 按需 | `references/samples/` 中的对应样本  | 先看 INDEX.md 选择，不全部加载       |

### 4.2 文体档案对照表

| 文体          | 读取文件                           |
| ----------- | ------------------------------ |
| 学术论文        | `references/style-paper.md`    |
| 公众号文章       | `references/style-wechat.md`   |
| 课题研究方案/课题本子 | `references/style-proposal.md` |
| 工作方案/汇报/报告  | `references/style-workplan.md` |

### 4.3 声音地基加载规则

**模式 A（代写生成）：** 每次必须执行声音地基加载（见 `references/modes.md` 第2.5步），无论长短文。从样本库按功能类型选 2-3 段原文 + 从改写对照库选 8-10 对，直接嵌入生成和评审智能体的 prompt。

**模式 B（润色改写）：** 若润色文本超过 800 字，或润色后感觉"风格对了但神不对"，按第2.5步的选样表选 1-2 段原文加载，作音调校准。短段落润色（500字以内）可跳过。

**模式 C（风格迁移）：** 重点在两个文体档案的对比，通常不需要加载样本。若目标文体的声音感觉不确定，可选 1 段目标文体的原文段落作参照。

**改写对照库（`references/pattern-pairs.md`）：** 模式 A/B/C 中，若执行 S1-S5 风格扫描后仍觉改写方向不确定，可参照此文件的具体对照对。
