# 风格档案 — 学术论文

- **文体名称：** 学术论文（中文）
- **建档日期：** 2026-03-23
- **档案版本：** v6.2（按子结构重构）
- **前置依赖：** `style-core.md`（必须先读 core，再读本档案）

---

> **本档案的定位**
> 以下是学术论文中常见的子结构及其写法规律。
> 根据具体论文类型和内容需求选用，不必全部包含。未覆盖的结构按 core 层规则 + 内容逻辑自行组织。
>
> 读取顺序：先读 core → 再读本档案。core 与本档案冲突时，以本档案为准（文体特化优先）。
>
> **常见论文类型与子结构组合参考**（菜单，不是模板）：
> - 理论+技术型：引言→综述→理论框架→技术方法→案例验证→结论讨论
> - 技术应用型（短文）：引言→技术方法→案例验证→结语
> - 理论分析型：引言→理论分析×多节→案例验证→结论讨论
> - 工程系统型：引言→（综述→）系统架构→技术方法→案例验证→结论讨论

## 一、通用规范（所有子结构共享）

### 1.1 人称与语气

- 全程"本研究""本文""研究结果表明"，不用第一人称"我""我们"
- 正式但不冷漠，技术性严肃感是底色
- 不用感叹号和反问句制造情绪
- 不用"我认为/我们觉得"式主观断言，论断通过数据、文献或逻辑推导承载
- 叙事者保持隐身

### 1.2 句子构建

- **手段-目的嵌套**是基本句型：句子自然呈现"通过X实现Y""以X为基础构建Y""基于X解决Y"的结构，这是目的性思维的语言映射，不是刻意套模板
- **长句用于推论**（需承载多个逻辑步骤），**短句用于结论和定义**（需清晰有力），长短变化由逻辑层次决定
- 极少使用比喻，一旦使用必然在关键处，且紧跟逻辑链条来实化比喻

### 1.3 段落构建

- 每段是一个完整的论证单元：论点（通常在段首）+ 支撑展开 + 逻辑收束
- 段落不是话题的展开，而是论点的论证；读者看完段首句，应能预判本段要证明什么
- 段落末尾点明本段的逻辑归宿或与下文的连接，不让内容自然结束
- **节奏**：每2-3段为一组，末尾以"总体而言/综上/总之"收束
- **反过拟合警告**：以上是对作者自然写作习惯的观察，不是刚性模板。若连续三段以上均呈现相同的"首句定性+中间展开+末句收束"结构，说明在套模板而非在写作，必须打破节奏——可以从具体现象切入再上升到论点，可以让段首直接承接上段逻辑而不另起定性。结构由内容逻辑决定，不由格式惯例决定

### 1.4 用词倾向

- 大量使用四字格结构："前所未有""深刻重塑""互为补充""因地制宜""提质增效""多维整合""动态适应"等
- 偏好"XXX为XXX提供了XXX"的贡献声明句式
- 动词偏好：构建/提出/探索/实现/支撑/推动/聚焦/围绕/赋能
- 连接词偏好：有鉴于此、鉴于此、为此、是以、由此可见、综上所述、总体而言、总之
- 术语首次出现时中文名+英文全称+缩写："人类反馈强化学习（reinforcement learning from human feedback，RLHF）"

### 1.5 声音质地（朴素正式感）

- **务实落地，不概念拔高**：当一个事物可以用"成为主要数据源之一"描述时，不写"催生了技术路线"。选择把事物放回它实际扮演的角色，而非给它一个更宏大的叙事定位
- **能删的修饰词一律删**：单个装饰性形容词也要审视。"基础性描述"→"基础描述"，"结构性能力局限"→"能力局限"。判断标准：去掉这个词之后语义是否有实质损失，无损失则删
- **选词精确优先于选词有力**："构成"比"构建"更适合表示静态组成关系，选对的那个，不选响亮的那个
- **并列项用短语节奏压缩**：三个以上的并列描述，优先压缩为节奏对称的短语组（"采集手段各异、存储平台不同、数据标准不一"），而非每项展开为完整从句
- **用"如何…"设问引出挑战**：框定问题时优先用"如何将其融合为……是先决条件"的设问结构，而非"核心挑战是……"的直接陈述
- **段间过渡用内容链接，不用结构标记**："在数据采集的基础上，往往还面临……"优于"进一步，……同样不容忽视"。前者用上文内容搭桥，后者靠副词强行推进
- **转入问题前先正面铺垫**：指出某个领域的不足或挑战之前，先用一句话承认现有成果或现状，再自然过渡到问题

### 1.6 文献引用规则

- 引用不堆砌，每处引用至少有一个明确功能：①建立研究基础 ②为主张提供权威支撑 ③精准指出现有研究局限
- 功能不明确的引用不写
- 引用密度分布：引言 > 综述 > 方法 >> 案例（案例几乎无引用）
- **AI 代写场景补充规则**：生成的参考文献信息须可查证。核心引用的作者、期刊/会议名称、年份须准确；无法确认的引用不写入正文

### 1.7 论证质量要求

> 本节规则面向**生成智能体**，在写作过程中执行。与评审标尺（`review-rubric-paper.md`）形成互补——本节是"怎么写出好的论证"，评审标尺是"怎么判断论证好不好"。

**1.7a 论证 vs 描述 vs 同义反复——写作时的自判标准**

每写完一个核心段落，自问：这段在做什么？
- **论证**：给出了因果推理链——因为 A（有证据），所以 B。读者看完能被说服
- **描述**：交代了"是什么""有哪些"，但没回答"为什么"或"凭什么"。描述段允许存在，但不能出现在论证的承重位置
- **同义反复**：结论和前提换了一种说法表达同一件事。典型信号："X 很重要，因为 X 对 Y 来说是关键的"——"重要"和"关键"意思相同，论证链条为零。同义反复**在任何位置都不允许**

**1.7b 缺口精确定位**

指出现有研究不足时，缺口须定位到**具体维度**，不泛泛说"研究不足""尚待深入"。

- 好："当前主流大模型缺乏对地理空间信息的**内生表达与推理机制**"——缺口锁定在"空间推理机制"
- 差："当前研究仍存在诸多不足"——什么不足？在哪个层次不足？不清楚

**1.7c 论证上行——末句推进而非重复**

论证段落的收束句应将论点推进到更高层次，而非重复本段已说的内容。

- 好："从这个意义上说，垂直大模型的构建既是技术工程，更是认知工程与制度工程"——从技术层面升维到方法论层面
- 差："因此，上述三个层次构成了垂直大模型的核心特征"——只是把本段内容复述了一遍

### 1.9 代写规划扩展（Mode A 专用）

> 执行 modes.md 模式 A 时，在完成通用结构规划（四问）之后，补充执行以下论文专属规划项。

**评审机制配置：**

| 项目 | 配置值 |
|------|--------|
| 评审标尺文件 | `references/review-rubric-paper.md` |
| 启用条件 | 正文字数 ≥ 2000 字（学术论文通常满足） |
| 评审角色 | 见评审标尺文件顶部"评审定位"部分（博士生导师视角） |

**规划补充项（完成通用四问后依次确认）：**

- **原创贡献定位**：本文提出的新东西具体是什么？用一句话陈述。无法用一句话说清"新在哪" → 重新思考贡献点，不进入生成阶段
- **逐节认知增量**：每一节要让读者获得什么之前不知道的认知？若某节的预期增量仅为领域常识 → 规划阶段就须想清楚该节如何在常识之上构建新的分析层次
- **关键承重段**：全文哪 2-3 个段落是论证链条的承重点？这些段落需要什么级别的论据（定量数据 / 权威文献结论 / 定性分析）？承重段论据不足 → 返回素材准备，不进入生成阶段

### 1.8 反模板检测（扩展）

在 1.3 段落构建的"反过拟合警告"基础上，增加**子节级检测**：

- **段落级**（1.3 已有）：连续三段以上呈现相同的"首句定性+中间展开+末句收束"结构 → 必须打破
- **子节级**（新增）：连续两个以上子节遵循完全相同的叙事模式 → 至少一个子节须改变切入角度或叙事顺序。典型违规信号：
  - 每个场景分析都是"痛点描述→路径推荐→组合建议"三段式
  - 每条技术路径都是"定义→原理→行业意义"三段式
  - 打破方式：从案例或数据切入再上升到原理、以问题而非定义开头、合并两个相似子节进行对比分析

## 二、引言/研究背景

### 段落逻辑：宏观背景 → 问题界定 → （可选：简要学者回顾） → 本文工作声明

**第一段：宏观背景铺垫（由大及小的聚焦漏斗）**
- 开局以宏观的时代/政策/技术趋势起笔，逐步聚焦到具体领域
- 开头句建立**时代/制度/技术坐标**（时间锚点或权威来源）
- 中间句展开趋势描述（"不仅……而且……""从……到……"等递进句式）
- 收束句通过**"然而/但"转折**引出局限或不足，自然过渡到第二段

**第二段：问题界定（缺口暴露）**
- 直接承接上段转折，明确指出"现有XX存在XX不足"
- 可并列式问题陈述（显式编号"(1)……(2)……(3)……"），也可概括性指出
- 问题与背景之间有因果链条，不凭空列举
- 常引入政策文件/权威引述增强问题的紧迫性
- 可用"尽管……但"句式，或"两层缺口"方式：先肯定现有成效，再转折指出差距

**可选插入：简要学者工作回顾**（当不设独立综述章节时）
- "近年来，相关学者围绕如何……进行了探索。例如，XX等提出……YY等……"
- 以"总体而言/总结来看"收束后再进入本文工作声明

**第三段：本文工作声明（研究宣言）**
- 以"有鉴于此"/"鉴于此"/"为此"/"针对……"等因果连接词引出
- 在一个大句中完成"做什么→怎么做→为什么做"的完整声明
- 以**"以期为……提供……"**收尾——"以期"是强共性收尾用语
- 通常只有1-3句，高度凝练

### 引言独有特征
- **密集引用政策文件**——引言是全文政策引用最密集的地方
- **"然而/但"转折是核心结构枢纽**——决定了后续所有章节的展开方向
- **不出现技术细节**——只讲"是什么问题""为什么重要"，不讲"怎么做"
- 引言编号方式不是强约束：可不编号（主流），也可用"0 引言"

## 三、文献综述（可选，可嵌入引言）

> 是否独立设章取决于论文类型和篇幅：综合性或篇幅较长的论文通常设独立综述章节，技术应用型短文多将综述嵌入引言或省略。

### 段落逻辑：分类综述 → 各类代表工作述评 → "综上…然而…亟需"缺口声明

**分类组织方式**
- 按地理/国别维度："国外→国内"二段式，或"总体渗透→国际→国内"三段式
- 按时间线：从早期到近年
- 按技术/主题维度："在XX方面……在YY领域……"

**单条文献叙述模式**
- "XX等[ref]聚焦/提出XX → 通过XX方法 → 实现/证明XX"
- 文献之间用"而""此外""与之相对"等轻量连接词衔接，或用"在XX方面""在XX领域"作分类引导

**综述收尾的固定套路**
- "综上所述/总体而言" + 已有工作的正面评价 + "然而/但" + 缺口指出 + "因此/亟需" + 本文的切入点
- 缺口声明与引言的缺口声明相互呼应但更具体——引言的缺口是宏观的，综述的缺口是技术性的

### 综述独有特征
- **高密度引用**——几乎每个判断都带参考文献
- **评述语气而非论证语气**——转述他人工作并评价，不需自己给出证据
- 综述对象可以是学术文献，也可以是政策实践和工具系统

## 四、理论框架/方法设计

> 方法章节有三种写法模式，根据内容性质选用。同一论文中可以混用不同模式。

### 模式A：概念阐释型

**段落逻辑：定义/定性判断 → 层层展开 → 归纳意义**
- 段首句：定义或定性判断（"XX是……"或"XX并非……而是……"）
- 中间句群：从抽象到具体（"具体而言……"），内容之间通过内容自然承接展开，不依赖编号路标词
- 段尾句：归纳意义或引出下一层

### 模式B：技术方案型

**段落逻辑：定义定位 → 通用技术分类 → 面向行业的特殊适配**
- 子节开头段：定义该技术是什么 + 在本研究中的定位
- 技术原理段：介绍主流方法与分类（常有2-3种分类或路径）
- 本文方案段：针对行业需求提出具体方案
- 多个技术路径或分类的展开，通过内容本身的逻辑递进自然衔接，不使用"其一/其二/其三"或"首先/其次/再者/最后"等编号路标词；每个分类或路径段落首句点明该路径的核心定位，由此实现隐性区分
- 过渡句常用"对于XX而言""面向XX需求""针对XX问题"
- **"先通用后行业"的二层叙事**——先讲通用技术原理，再讲"对于具体行业而言"如何适配

### 模式C：系统架构型

**段落逻辑：总述体系目标与构成 → 分项逐条展开各组件 → 总括句收束**
- 总述段概括整个体系的核心目标和基本构成
- 分项以编号列表"（1）……（2）……（3）……"逐项介绍各子模块，或按子节分设
- 可使用引号概念化命名作为高层框架标签（如"大算力""大数据""大模型""大场景"）
- 段首常用"XX是体系的……"功能定位句
- 总括句收束："总体而言" + "有赖于……协同联动/系统耦合"
- 重在描述各组件的功能定位和相互关系，不做深入技术原理解释
- 建设原则/设计理念的阐述多于技术细节

### 方法章节的共性特征
- **不使用"然而/但"做问题暴露**——方法章节是建设性的，不需要反复指出缺口
- **比喻密度最高**——用于降低技术概念的理解门槛（尤其在概念阐释和技术方案中）
- **引用模式变化**——综述中引用的是他人工作的结论，方法中引用的是技术的原始出处
- 每个子节末尾几乎都有一个"总体而言/综上"收束段

### 概念界定的固定手法

对核心概念的定义通常采用"否定式 + 正面界定 + 使命指向"三步：
- 先说"不是什么"（排除误解）
- 再说"是什么"（正面定义）
- 再说"为了什么"（功能或使命）

三个层次可在一句话内完成，不拆成三个独立句子。

## 五、案例/实验验证

### 段落逻辑：场景痛点 → 本文方案 → "以XX为例"具体展示 → 效果评价

**场景痛点描述**（每个子节开头）
- 直接点出该场景面临的传统困难，或界定应用范围
- 可用"针对……问题，通过……"的固定句式

**本文方案**
- 紧接说明"基于XX技术，构建/实现了XX"

**具体案例**
- **"以XX为例"是核心句式标志**
- 常配合"如图X所示"指向系统截图或结果对比图

**效果评价**
- **对比叙事**是核心修辞：传统方式 vs. 本文方案（"传统……往往存在……本文方案……"）
- 效果量化（时间缩短、精度提升等）增强说服力，根据需要选用，非必须；以定性描述为主亦可
- 总结性评价句收尾（"总体而言/整体上"）

### 案例独有特征
- **叙事而非论证**——按时间/操作流程顺序展开，更像"讲故事"
- **引用密度最低**——几乎没有参考文献引用，完全是自己的工作
- **配图最多**
- 方法与案例可以融合为一节（每个子节前半部分讲方法原理，后半部分讲案例），但分离型仍是主流写法

## 六、结论与讨论

### 结论部分

**两种格式均为常用写法，无强偏好**：
- **分条编号式**："本文围绕……主要结论如下：（1）……（2）……（3）……"——每条以核心判断开头（完整论断而非话题词），展开1-2句解释，最后给意义。条与条之间相互独立。篇幅较长或技术含量高的论文多用此式
- **综合叙述式**：以连续的叙述句回溯全文各部分贡献，不使用编号。篇幅较短或综合性论文多用此式

**结论开头的回溯概括**
- "本文围绕……系统探讨了……"/"面向……本文按照……提出了……"
- 与引言的"本文将做什么"形成闭合

### 讨论部分

**段落逻辑：承认局限 → 分维度展开挑战/展望 → 升华收束**

- **承认局限**："尽管……但""但由于……仍处在探索发展期"
- **分维度展开**：展开2-3个挑战/方向时，通过内容的自然递进衔接，每个方向段落首句点明核心，不使用"一方面……另一方面……"或"首先……其次……"等编号路标词
- 工程系统类论文的讨论可延伸到**制度/组织层面建议**（提出"技术共享+制度机制"的双轨建议方向）
- **升华收束**："可以预见""必将""将成为"——展望标志词
- "综上/总之"是讨论部分的最终收束信号

### 结论讨论独有特征
- **回溯式写法**——重新概括"本文做了什么"，与引言形成首尾呼应
- **不引入新的技术内容**——只对已有内容做总结和升华
- **从具体到抽象的逆向运动**——从具体结果抽象为意义
- 语气从陈述转向展望——"将""必将""可以预见"等将来时态标记

## 七、摘要（压缩结构）

摘要 = ①一句话说明研究背景价值 → ②精确指出现有问题 → ③逐条技术贡献（每条"基于X，实现Y"） → ④一句话说明整体贡献。压缩但不跳步骤。

---

## 八、稳定禁忌（论文特有）

> 以下禁忌仅在学术论文文体中额外生效，与 core 层禁忌叠加执行。

- **禁止文学性开场**：不从故事、场景、个人感受切入，始终从现状/问题/研究目标切入
- **禁止主观评价词**：不说"令人振奋的成果""具有划时代意义"等
- **禁止情感渲染词**："非常重要""极其关键"等不出现在学术论文中
- **禁止感叹号和反问句**：学术写作中这两类标点均属情绪渲染
- **禁止断裂式引用**：不出现"文献[1-5]表明……"这种把多篇文献笼统打包的引用方式，除非这些文献确实在同一个论点上形成共识

---

## 九、领域防火墙

论文内容涉及的专业领域知识（国土空间规划、自然资源管理等）属于样本的主题层面，不属于风格层面。写其他领域的论文时，只迁移本档案的写法规律和风格特征，不迁移领域知识和术语。
