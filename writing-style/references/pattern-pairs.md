# 改写对照库

作者本人对 AI 生成内容的实际改写习惯，按类别整理为"AI 默认写法 → 作者改法"对照。

**两个用途：**
1. **生成时注入**：主控按文体和任务类型选取最相关的 8-10 对，原文附在生成智能体 prompt 中
2. **扫描时参照**：S1-S5 风格扫描时，用此文件的对照印证修改方向

**选对原则：**
- 第一类（标点陷阱）的前三对，每次必传，不论文体
- 其余按文体和任务类型选：学术论文侧重第二、五类；公众号侧重第二、六类；课题方案侧重第二、三类

---

## 第一类：标点陷阱（每次必传）

**P01 冒号 + 完整句（最典型 AI 腔句式）**

```
[AI]: 核心问题：如何在精度与泛化之间找到平衡？
[改]: 核心问题是如何在精度与泛化之间找到平衡。

[AI]: 这意味着：系统必须具备动态调整能力。
[改]: 这说明系统必须具备动态调整能力。

[AI]: 研究发现：准确率在三个场景下均超过 90%。
[改]: 研究发现，准确率在三个场景下均超过 90%。
```

> 判断标准：冒号后若接完整句子，一律改为逗号衔接或重组句子。"X：完整句"是零容忍格式。

---

**P02 破折号作强调（非必要不用）**

```
[AI]: 这种模式——高集成、低耦合——在工程实践中已被广泛验证。
[改]: 这种高集成、低耦合的模式在工程实践中已被广泛验证。
（若需强调对比，改为：这种模式在工程实践中已被广泛验证，集成度高、模块间耦合低。）
```

> 判断标准：去掉破折号并重组，语义是否完整？完整则去掉。仅在插入说明且逗号无法胜任时保留。

---

**P03 括号融入正文**

```
[AI]: 相比基线模型（ResNet-50），本方法精度提升了 8 个百分点。
[改]: 相比基线模型 ResNet-50，本方法精度提升了 8 个百分点。

[AI]: 该分析方法（下文简称"动态融合法"）在三类数据集上进行了验证。
[保留]: 首次引入缩写，括号可保留。
```

> 判断标准：括号内的信息能融入正文则融入。缩写首次定义、非必读辅助信息可保留。

---

## 第二类：AI 特征句式（声音差距最大）

**P04 "不是 A 而是 B"（高频 AI 句式）**

```
[AI]: 这不是技术局限，而是数据稀缺的问题。
[改]: 问题出在数据稀缺上。（读者不需要被纠正"技术局限"这个认知时，直接说 B）

[AI]: 这不是单纯的效率优化，而是一次范式层面的转变。
[改]: 这是一次范式层面的转变，不只是效率优化。（若需对比，换一个方向说）
```

> 判断标准：删掉"不是 A"，语义是否完整？完整则删，只说 B。
> 白名单：读者确实持有错误认知 A 时，纠正有修辞价值，可保留。

---

**P05 "这意味着"（去掉，直接说）**

```
[AI]: 这意味着，研究者需要重新审视评估框架的设计原则。
[改]: 评估框架的设计原则因此需要重新审视。

[AI]: 上述发现意味着现有方法在长尾分布下存在系统性偏差。
[改]: 现有方法在长尾分布下存在系统性偏差。
```

> 判断标准："这意味着"是元评论壳子，去掉后直接说结论更有力。

---

**P06 首先 / 其次 / 最后（路标词充当衔接）**

```
[AI]: 首先，本文构建了分析框架。其次，基于该框架进行了实证验证。最后，根据结果提出了政策建议。
[改]: 本文构建了分析框架，基于该框架进行了实证验证，并根据结果提出了政策建议。
```

> 判断标准：去掉路标词后，读者是否还能感受到递进关系？能则去掉。
> 例外：政府公文等正式文体中，"其一/其二"可偶尔使用。

---

**P07 一方面 / 另一方面（并列路标词）**

```
[AI]: 一方面，该方法提高了计算效率；另一方面，精度有所下降。
[改]: 该方法在提高计算效率的同时，精度有所下降。
```

---

**P08 综上所述 / 总之（宣布要总结了）**

```
[AI]: 综上所述，本方法在准确率、鲁棒性和可解释性三个维度上均优于基线。
[改]: 本方法在准确率、鲁棒性和可解释性三个维度上均优于基线。

需要收束信号时，换"由此"/"因此"，不用"综上所述"。
```

## 第三类：修饰词膨胀（容易被忽视）

**P09 结构性 / 系统性 / 战略性（万能修饰词）**

```
[AI]: 这是一次结构性的范式转变。
判断：去掉"结构性的"后，"这是一次范式转变"语义完整 → 删。

[AI]: 构建系统性的技术方法体系。
[改]: 构建技术方法体系。（"系统性的"几乎从不增加有效信息）
```

> 判断标准：去掉修饰词后语义有无实质损失？无损失则删。仅在与"表面性""局部性"等有真实对比时保留。

---

**P10 重要的是 / 值得注意的是（元评论）**

```
[AI]: 重要的是，这一机制在极端条件下仍然有效。
[改]: 这一机制在极端条件下仍然有效。

[AI]: 值得注意的是，样本量对结果有显著影响。
[改]: 样本量对结果有显著影响。
```

> 判断标准：这类词从不增加信息，直接删。

---

**P11 深入 / 全面 / 系统（虚词程度修饰）**

```
[AI]: 本文对现有方法进行了深入分析。
[改]: 本文从 X 和 Y 两个维度分析了现有方法。（把"深入"落实为具体维度）

[AI]: 建立全面的监测体系。
[改]: 建立覆盖 X、Y、Z 三类指标的监测体系。（把"全面"落实为范围）
```

---

## 第四类：指示代词累积（细节，高频）

**P12 "这一"高频（重复核心名词）**

```
[AI]: 提出了动态融合策略。这一策略在三个场景下均表现出色。
[改]: 提出了动态融合策略，在三个场景下均表现出色。（合并为一句）
或：提出了动态融合策略，该策略在三个场景下…（"该"比"这一"更简洁）
```

> 判断标准：全文"这""这一""这个""它"合计超过 3 处即须削减，优先合并或直接重复核心名词。

---

**P13 "这使得"（换直接描述）**

```
[AI]: 数据稀缺，这使得模型在长尾场景下性能显著下降。
[改]: 数据稀缺，模型在长尾场景下性能显著下降。
或：数据稀缺导致模型在长尾场景下性能显著下降。
```

---

## 第五类：论证腔（学术写作特有）

**P14 同义反复（最常见论证缺陷）**

```
[AI]: 该方法至关重要，因为它在整个技术链条中发挥着关键作用。
问题："至关重要"和"关键作用"是同一件事，没有论证链。
[改]: 该方法决定了后续链条中 X 和 Y 两步的有效性，若缺失则 Z 无法执行。
```

---

**P15 结论段"具有重要意义"（空洞收束）**

```
[AI]: 本研究对推动行业智能化发展具有重要的理论意义和实践意义。
[改]: 本研究为行业智能化提供了一套可迁移的评估框架，尤其适用于数据稀缺的行业场景。
（把意义说具体：提供了什么、适用于哪里、有什么限定条件）
```

> 判断标准：删掉"具有重要意义"这句，读者会不会损失任何具体信息？不会损失则删，会损失则说清楚具体是什么意义。

---

**P16 概念悬空（引入但不界定）**

```
[AI]: 本文引入了"场景感知融合"的概念，并在三类数据集上进行了验证。
[改]: 本文引入"场景感知融合"的概念，指系统在运行时动态识别当前数据场景并自适应调整融合策略的能力，在三类数据集上对其有效性进行了验证。
```

---

**P17 分析元评论（去掉壳，直接说发现）**

```
[AI]: 通过对 2000-2020 年数据的分析，本文发现积雪面积呈现显著下降趋势。
[改]: 2000-2020 年间，积雪面积呈显著下降趋势。
（"通过对…的分析，本文发现"是元评论壳，读者需要的是发现，不是分析过程的声明）
```

---

## 第六类：公众号特有模式

**P18 长解释句（打短）**

```
[AI]: 这种在不同场景之间自适应切换、根据上下文动态调整参数的能力，正是传统静态模型所不具备的核心竞争力。
[改]: 传统静态模型没有这个能力——根据场景切换、实时调参。这才是核心差距。
（打断为 2-3 句，用短句承接关键词，不要一句塞完）
```

---

**P19 段首"此外 / 另外 / 同时"（路标词）**

```
[AI]: 此外，这个方法还解决了响应延迟问题。
[改]: 响应延迟问题也在这里得到了解决。
（公众号用内容本身承接，不用路标词开头）
```

---

**P20 段尾总结句重复论点（去掉）**

```
[AI]: [展开了三种场景分析后] 由此可见，场景感知融合在多种场景下均展现出优越的适应性。
问题：段尾这句只是重复了段落标题，没有新信息。
[改]: 删掉。或用一句点睛句收束——不是复述已有内容，而是提炼出一个共同规律或指向下一步的钩子。
```
